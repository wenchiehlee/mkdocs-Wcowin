> [如何將 github pages 遷移到 vercel 上託管](https://cloud.tencent.com/developer/article/1771693?from=article.detail.2065884&areaSource=106000.11&traceId=URfJ8WWU7TMKAua9v9kI1)


<h2 id="%E5%89%8D%E8%A8%80" name="%E5%89%8D%E8%A8%80"><strong>前言</strong></h2><p>早期網站使用 <code>github pages</code>,後來遷移到 coding,最近又放到騰訊雲網站靜態託管,無論是 coding 的 cos 儲存桶,還是<a data-text-link="143_1771693" data-from="10680" href="https://cloud.tencent.com/product/wh?from=10680" target="_blank">靜態網站託管</a></p><p>他們都是收費的,那有沒有免費的託管商呢,既不影響網站的訪問速度還免費,於是,找了一下,還真有,vercel和Netlify,就是免費的</p><p>其中大名頂頂的<code>Next.js</code>,<code>create-react-app</code>,<code>Nuxt.js</code>等就是部署在部署託管在<code>vercel</code>的,而<code>vuejs</code>,<code>reactjs</code>等就是託管在<code>Netlify</code>上的</p><p>想必經常開啟這些鼎鼎大名官網的時候,訪問速度還是可以的</p><p>今天說一下這個<code>vercel</code>,Vercel 可以部署任何前端應用程式的最佳場所。號稱以零配置部署到我們的全球邊緣網路,動態擴充套件到數百萬個頁面而不會費力</p><p>一鍵匯入(支援 github,gitlab,BitBucket),自己的儲存庫，然後推送。vercel 內建的<code>CI / CD</code>系統會在每次程式碼更改時觸發</p><p>體驗過後,確實方便,強大</p><h1 id="01" name="01"><strong>01</strong></h1><p><strong>為什麼選擇 vercel</strong></p><p>⒈ 免費部署託管前端應用</p><p>⒉ 支援一鍵匯入(github,gitlab),零配置</p><p>⒊ 支援自定義<a data-text-link="3020_1771693" data-from="10680" href="https://cloud.tencent.com/act/pro/domain-sales?from=10680" target="_blank">域名</a>以及配置 ssl 證書,https.</p><p>⒋ 簡單友好,個人賬戶免費,可以部署 next.js,Nuxt.js,Gatsby.js,Angular,Ember.js,Hexo,Eleventy,Hugo,Docusaurus2,Preact,Dojo,Vue.js,Jekyll 等</p><h1 id="02" name="02"><strong>02</strong></h1><p><strong>前提條件</strong></p><p>⒈ 先成功部署 github pages(本文以 github pages 為例,其他類似的應用也是如此)</p><p>⒉ 然後登陸 vercel 官網, 一鍵匯入</p><h1 id="03" name="03"><strong>03</strong></h1><p><strong>怎麼使用呢</strong></p><p>在自己的 github 倉庫中找到 github pages 倉庫</p><figure class=""><div class="image-block"><span><div class="" style="text-align: center;
